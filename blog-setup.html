我的第一个博客！

Date Tue 25 August 2015 By Ben Category Computer. Tags blog pelican markdown

Blog Setup

之前和同学一起学unity3D, 在Iteye上面写博客写了几篇就坚持不下来了。。。这几天突然想着要搭一个自己的博客。各种google，百度终于在框架搭好了，记录一下(中间ubuntu, windows两个系统各种折腾，终于都打好了一 一+)

Linux下的搭建

选用的工具是python + pelican + github 搭建一个静态博客， 参考网站很多，主要是这个GITHUB个人博客搭建-Pelican 在Windows环境下的安装及配置以及"用Pelican和Github Pages在Linux上搭建个人博客"。

首先你得有个github帐号， 官网申请完之后，建立一个username.github.io的版本库，然后把你的博客放进这个库中，然后就可以直接登录username.github.io的网址看见你的博客了

然后选用的框架是pelican, 看到说是基于python的我就直接选了这个了，反正是第一次。。 先把这个pip的安装程序弄下来get-pip.py

sudo apt-get install python   #安装Python
python get-pip.py             #安装pip
pip install pelican           #安装pelican
pip install markdown          #安装markdown
pelican-quickstart            #建立blog的框架

这是整个文件夹的结构

blog/
|--content #存储所写的blog文件目录，
|   └──（blog pages) #博客文件，如*.md后缀的文件
|--output #执行make publish之后，会将博客文件转换成相应的html文件及相应的存储目录
|   └──.git #执行git init之后，产生的远程仓库目录
|   └──author #作者目录，存放pelicanconf.py中指定的Author文件
|   └──category #分类目录，默认目录文件为misc，在编辑器中指定
|   └──feeds #目录，pelicanconf.py中FEED_RSS指定的XML文件
|   └──pages #目录，pelicanconf.py中ARTICLE_URL指定格式分类文件
|   └──tag #目录，存放编辑器开头所指定的Tag
|   └──theme #目录，存放博客模板文件
|   └──archives.html,authors.html,categories.html,index.html, tags.html
|——pelican-plugins #目录，pelican插件
|——develop_server.sh #pelican简易web服务器操作文件
|——fabfile.py # 与make类似，可用fab publish, fab server命令
|——Makefile #make 命令文件
|——pelicanconf.py #主要的pelican配置文件
|——publishconf.py #发布配置文件,可有可无

我们主要在content写下博客的内容， 在pelicanconf.py修改配置， makefile修改或添加github发布 markdown的语法参见Markdown语法简介。 接下来我们先选择一个Pelican主题样式, 从这里pelican样式 fork或者clone下来， 然后选择一个样式，比方说neat, 将其copy至blog目录下， 然后修改pelicanconf.py文件， 添加THEME = "neat"即可采用该样式了。

为了便于讲output中生成的博客直接发布到github中， 修改Makefile文件如下:

TARGETDIR = $(BASEDIR)/../mihawk2010.github.io
publish:
    $(PELICAN) $(INPUTDIR) -o $(OUTPUTDIR) -s $(CONFFILE) $(PELICANOPTS) 
github: publish
    cp -r -u $(OUTPUTDIR)/. $(TARGETDIR)/
    cd $(TARGETDIR) && git add --all
    cd $(TARGETDIR) && git commit -m 'commit'
    cd $(TARGETDIR) && git push origin master

我的做法比较怪， 因为我直接按照他们的方法改git会报错， 可能我有些设置不对把。这里我就自己慢慢测试成功是这个样子。 在这之前还要配置你的git身份，并把username.github.io远程仓库添加至本地，然后我们就可以利用

make github

直接发布博客了。

Windows下的配置

一开始我准备直接使用windows的， 类似的安装好python, pip, pelican, make, 并且也可以成功发布。 但是我想使用make devserver指令调试，这样可以实时的看到我的博客的变化， 这点在windows下做不到(不过我没试过fab)。 所以我就决定用Cygwin了。

安装Cygwin的时候， 可以要勾选python, git, make安装， 不然打开什么都有。 然后类似于Linux下的过程一步一步来就好了。 我遇到的折腾的问题就是路径不对， 因为我的Windows里面装好了python, pip, pelican， 然后在Cygwin里面执行这些指令就会调用windows下的这些程序， 但是这些程序不认Cygwin的路径。。搞了半天， 我把Windows下默认的python的环境变量都删了， 然后一步一步安装就ok了。

在Windows下写markdown用的是markdownPad2， 用起来很方便， 比linux下的retext好用。不过刚才手贱点了更新， 破解的就没了。。。 之后等别的功能也一起装好了，再来写吧(┬＿┬)
